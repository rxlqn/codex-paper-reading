# Open Source Checklist

发布到 GitHub 前建议确认这些事项。

## 内容边界

- [ ] 不提交 `.DS_Store`、临时文件和导出产物。
- [ ] 不提交第三方论文 PDF，除非逐篇确认许可证允许再分发。
- [ ] 对论文结论、图表和公式只做必要引用和转述，不大段复制原文。
- [ ] 所有笔记都保留论文链接和本地来源路径。
- [ ] 从第三方项目改写的 workflow / skill 保留 source URL、upstream license、必要 notice 和改写说明。
- [ ] 小红书草稿如果含有未发布观点，发布前再移动到 `published/`。

## 仓库元信息

- [ ] 选择仓库名。
- [ ] 选择许可证。
- [ ] 确认是否公开 `shares/xiaohongshu/drafts/`。
- [ ] 确认是否需要英文 README 摘要。

## 推荐许可证策略

这个仓库里有三类内容，许可证最好分开考虑：

- 原创笔记、分享稿、主题综述: 可以考虑 `CC BY 4.0` 或 `CC BY-NC 4.0`。
- 模板和目录结构: 可以考虑 `MIT`。
- 第三方论文 PDF、论文图表: 默认不要再分发，只保留外部链接。

如果只想简单处理，建议先不提交 PDF，再给原创文字内容选择一个 Creative Commons 许可证。
